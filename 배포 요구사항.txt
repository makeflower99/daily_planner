# Daily Planner 배포 — 요구사항 명세

## 배경

사용자는 현재 Daily Planner 앱을 집 PC의 Docker 컨테이너에서 실행하고, Cloudflare TryCloudflare로 임시 URL을 통해 외부 접속하고 있다. 여기서 "배포하고 싶다"는 막연한 생각이 생겼고, 이를 질문을 통해 아래와 같이 구체화했다.

## 확정된 요구사항

### 1. 목적 (왜)
배포를 통해 아래 3가지를 동시에 해결하고 싶다.
- 접속 안정성: 현재 Cloudflare 임시 URL은 재시작할 때마다 바뀌어 불편하다. 항상 같은 주소로 접속하고 싶다.
- 지인 공유: 본인 외에 지인 10~30명이 사용할 수 있게 하고 싶다.
- 모바일 사용성: 스마트폰에서 앱처럼 쓸 수 있으면 좋겠다.

### 2. 사용자 규모 (누가)
- 지인 범위 10~30명.
- 불특정 다수 공개 서비스가 아니다.
- 회원가입 기능은 추가하지 않는다. 지금처럼 본인이 CLI로 직접 유저를 생성하고, 지인에게 아이디/비밀번호를 1:1로 전달하는 방식을 유지한다.

### 3. 사용 형태 (어떻게)
- PC/스마트폰 모두 웹 브라우저로 접속.
- 스마트폰에서는 홈화면에 아이콘으로 추가해서 앱처럼 쓰고 싶다.
- 네이티브 앱(앱스토어 배포, React Native, Electron 등)은 원하지 않는다.

### 4. 호스팅 위치 (어디에)
- 집 PC를 계속 서버로 사용한다.
- VPS, 라즈베리파이, 클라우드 호스팅으로 이전하지 않는다.
- 집 PC가 24시간 켜져 있다는 것을 전제로 한다.

### 5. 도메인과 비용
- 도메인을 신규로 구입할 의향이 있다.
- 예산은 연 1~2만원 수준.
- 고정 도메인 + HTTPS가 필요하다 (스마트폰 홈화면 추가는 HTTPS 고정 주소가 있어야 제대로 작동하기 때문).

---

## 배포 범위에 포함되는 것

- 고정 도메인 구입과 연결
- 집 PC에서 해당 도메인으로 안정적으로 접속되는 HTTPS 경로 확보
- 스마트폰 홈화면에 아이콘으로 추가 가능하게 만드는 최소한의 구성 (아이콘, 앱 이름, 전체화면 실행)
- 지인에게 전달할 URL 및 간단한 "홈화면 추가 방법" 안내

## 배포 범위에서 제외되는 것

- 회원가입 UI, 이메일 인증, 비밀번호 찾기
- 네이티브 앱 (앱스토어 배포)
- 푸시 알림
- 오프라인 사용
- VPS/클라우드 이전
- 약관/개인정보처리방침/어뷰징 대응 (지인 범위 비공개 사용이라 불필요)

---

## 이 작업과 분리해야 할 별개 이슈

이전 대화에서 드러난 Docker 볼륨 DB와 로컬 DB가 분리되어 있는 문제는 배포 요구사항과는 독립적이지만, 배포 후 실수로 잘못된 쪽을 켜면 "데이터가 사라졌다"는 지인의 오해를 부를 수 있다. 배포 실행에 착수하기 전에 어느 쪽 DB를 정본으로 쓸지 먼저 확정해둘 필요가 있다.
