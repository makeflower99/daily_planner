## 1. 프로젝트 개요

### 기술 스택

| 구분 | 내용 |
|------|------|
| 프론트엔드 | HTML5, CSS3, JavaScript (ES6+), Vanilla JS |
| 백엔드 | Node.js + Express |
| 데이터베이스 | SQLite (better-sqlite3) — 유저별 JSON blob |
| 인증 | express-session + bcrypt (서버 세션) |
| 보안 | helmet, express-rate-limit, session regeneration |
| 컨테이너 | Docker + Docker Compose |
| 외부 접속 | Cloudflare Tunnel (cloudflared, HTTP/2) |
| UI 언어 | 한국어 |

### 폴더 구조

```
daily_planner/
├── index.html              # 메인 HTML - 전체 페이지 구조 + 모달 + 초기화 스크립트
├── login.html              # 로그인 페이지
├── server.js               # Express 서버 (인증, API, 정적 파일)
├── db.js                   # SQLite DB 모듈 (유저/데이터 CRUD)
├── backup.js               # DB 자동 백업 (6시간 간격, 최근 7개 보존)
├── create-user.js          # 유저 생성 CLI 도구
├── package.json            # npm 의존성 정의
├── Dockerfile              # Docker 이미지 빌드 (Node 20 Alpine)
├── docker-compose.yml      # 서버 + Cloudflare Tunnel 통합 실행
├── .env                    # 환경변수 (SESSION_SECRET) — Git 제외
├── .env.example            # .env 템플릿 (Git 포함)
├── .dockerignore           # Docker 빌드 제외 파일
├── tunnel.bat              # Cloudflare Tunnel 실행 스크립트 (로컬용)
├── memo.txt                # 향후 기능 아이디어 메모
├── architecture.html       # 시스템 아키텍처 인터랙티브 문서
├── css/
│   └── styles.css          # 전체 스타일시트 - 다크/라이트 테마
├── js/
│   ├── data.js             # 데이터 관리 모듈 - 서버 API 연동, 로컬 캐시
│   ├── checklist.js        # 체크리스트 페이지 모듈 - 일일 체크 + 미니 캘린더
│   ├── calendar.js         # 캘린더 페이지 모듈 - 월간 뷰, 섹션/항목/식단 관리, 드래그앤드롭
│   └── tracker.js          # 일일 기록 페이지 모듈 - 계획 vs 실행 비교
└── backups/                # DB 백업 파일 저장소 — Git 제외
```

### 외부 의존성

**프론트엔드:**
| 의존성 | 용도 |
|--------|------|
| Google Fonts - Noto Sans KR (300, 400, 500, 700) | UI 전체 폰트 |

**백엔드 (npm):**
| 의존성 | 용도 |
|--------|------|
| express | 웹 서버 프레임워크 |
| better-sqlite3 | SQLite DB 드라이버 |
| better-sqlite3-session-store | 세션 SQLite 저장 |
| bcrypt | 비밀번호 해싱 |
| express-session | 세션 관리 |
| express-rate-limit | 로그인 브루트포스 방어 |
| helmet | 보안 HTTP 헤더 |
| morgan | 요청 로깅 |

### JS 로드 순서 및 초기화

```
data.js → checklist.js → calendar.js → tracker.js
```

페이지 로드 시 `loadData()` (async, 서버 fetch) → `.then()` → `initChecklist()` → `initCalendar()` → `initTracker()` 순서로 초기화.  
페이지 닫기 전(`beforeunload`) `navigator.sendBeacon()` POST로 서버에 데이터 전송.

## 2. 화면 명세서 + 페이지별 구현 기능 상세

### 네비게이션

상단 고정(sticky) 네비게이션 바에 3개 탭 버튼:

| 탭 | 아이콘 | 페이지 ID | 기본 상태 |
|----|--------|-----------|-----------|
| 체크리스트 | ✅ | `page-checklist` | **활성 (기본)** |
| 캘린더 | 📅 | `page-calendar` | 비활성 |
| 일일 기록 | 📊 | `page-tracker` | 비활성 |

탭 전환 시 캘린더/트래커 페이지는 자동 리렌더링됨.

### 2.1 체크리스트 페이지 (✅)

> 파일: `js/checklist.js` (219줄)  
> 용도: 선택한 날짜의 루틴을 체크하며 일일 진행률을 확인하는 메인 화면

#### UI 구성요소

```
┌─────────────────────────────────────┐
│  ‹  2026년 4월 3일 (금)  ›          │  ← 날짜 네비게이션 (클릭 시 미니 캘린더)
├─────────────────────────────────────┤
│  [미니 캘린더 팝업] (토글)           │  ← 월 단위 달력에서 날짜 선택 가능
├─────────────────────────────────────┤
│  오늘의 루틴                        │
│  진행률            6 / 12           │
│  ████████░░░░░░░  (프로그레스 바)    │
├────────┬────────┬───────────────────┤
│ 완료 6 │ 남은 6 │ 달성률 50%        │  ← 통계 카드 3개
├────────┴────────┴───────────────────┤
│  📝 메모                    저장됨 ✓│  ← 메모 입력 (자동 저장)
│  [오늘의 메모를 작성하세요...]       │
├─────────────────────────────────────┤
│  ● 아침 루틴  [매일]                │  ← 섹션 헤더 (색상 점 + 제목 + 반복 배지)
│  ☑ 🪥 양치                         │
│  ☐ 💧 따뜻한 물 마시기              │  ← 체크박스 항목들
│  ...                                │
├─────────────────────────────────────┤
│  ● 식단 계획                        │  ← 식단 섹션 (오렌지 테마)
│  🌅 아침                            │
│  ☐ 현미밥  ☐ 두부                   │  ← 식단 체크 항목
│  ...                                │
├─────────────────────────────────────┤
│  📥 데이터 내보내기 │ 📤 데이터 가져오기 │  ← 데이터 관리 버튼
└─────────────────────────────────────┘
```

#### 구현된 기능 상세

**1. 날짜 네비게이션**
- 좌우 화살표 버튼(`‹` `›`)으로 ±1일 이동
- 날짜 텍스트 클릭 시 미니 캘린더 팝업 토글
- 날짜 형식: `2026년 4월 3일 (금)`
- 오늘 날짜일 때 타이틀 "오늘의 루틴", 다른 날짜일 때 "루틴"으로 변경

**2. 미니 캘린더**
- 7x6 그리드 월간 달력 팝업
- 월 이동 (`‹` `›`) 지원
- 오늘 날짜 하이라이트 (보라색 텍스트, 볼드)
- 선택된 날짜 하이라이트 (보라색 배경)
- 이전/다음 월 날짜는 흐리게 표시
- 날짜 클릭 시 해당일로 체크리스트 전환 + 팝업 닫힘

**3. 섹션 필터링 및 렌더링**
- `getSectionsForDate(dateKey)` 호출하여 해당 날짜에 활성화된 섹션만 표시
- 반복 스케줄(매일/주중/주말/매주/격주/격일/매월) + 시작일/종료일 기반 필터링
- 섹션 없는 날짜: "이 날짜에 해당하는 체크리스트가 없습니다." 안내 메시지
- 각 섹션은 색상 점 + 제목 + 반복 주기 배지로 구성

**4. 체크박스 토글**
- 체크박스 또는 항목 라벨 클릭 시 완료 상태 토글
- 체크 시: 체크박스에 섹션 색상 배경 적용, 체크마크(✓) SVG 표시, 텍스트 취소선 + 투명도 50%
- 토글 시 즉시 `completions[dateKey][itemId]` 갱신 → `saveData()` 호출
- `saveDailySnapshot(dateKey)` 호출하여 일일 기록에 스냅샷 저장
- 캘린더가 활성 상태이면 자동 동기화 (`syncCalendar()`)

**5. 진행률 & 통계**
- 프로그레스 바: 완료 수 / 전체 수 기반 너비 퍼센트 (0.4s 애니메이션)
- 통계 카드 3개: 완료(보라), 남은 항목(오렌지), 달성률(틸)
- **식단 항목도 전체 통계에 포함** (체크리스트 + 식단 합산)

**6. 메모**
- 달성률 통계 카드 하단에 메모 입력 영역 배치
- `textarea`로 편집 가능, 입력 시 500ms 디바운스 자동 저장
- "저장됨 ✓" 피드백 표시 (1.5초 후 페이드 아웃)
- 캘린더 탭의 메모와 동일한 `data.memos[dateKey]` 데이터 공유
- 날짜 변경 시 해당 날짜 메모로 자동 전환

**7. 식단 표시 & 체크**
- 캘린더 페이지에서 등록한 식단을 읽기 전용으로 표시
- 끼니별(아침/점심/저녁/간식) 서브 라벨로 그룹화
- 체크박스로 식단 실행 여부 토글 가능 (오렌지 색상 체크)
- 식단이 없을 경우: "캘린더 탭에서 식단을 추가하세요." 안내

**8. 데이터 내보내기/가져오기**
- **내보내기**: 현재 데이터를 JSON 파일로 다운로드
  - 파일명: `planner_backup_YYYYMMDD.json`
  - 내보내기 전 오늘 스냅샷 자동 저장
- **가져오기**: JSON 파일 업로드 → 검증 → 전체 데이터 교체
  - `sections` 필드 존재 여부 검증
  - 성공 시 alert 알림, 전체 페이지 리렌더링
  - 실패 시 에러 메시지 alert

### 2.2 캘린더 페이지 (📅)

> 파일: `js/calendar.js` (952줄)  
> 용도: 월간 캘린더 뷰 + 섹션/항목 관리 + 식단 계획 + 메모

#### UI 구성요소

```
┌─────────────────────────────────────┐
│  ‹       2026년 4월       ›         │  ← 월 네비게이션
├─────────────────────────────────────┤
│  일  월  화  수  목  금  토          │
│  ★                                  │  ← ★ = 식단 있는 날
│  [1] [2] [3] [4] [5] [6] [7]       │  ← 숫자 + 색상 점 인디케이터
│  ●●  ●●  ●●  ...                   │  ← 섹션 색상 점
│  ...                                │
├─────────────────────────────────────┤
│  📋 등록된 루틴                      │  ← 섹션 관리 아코디언 패널
│  ┌─────────────────────────────┐    │
│  │ 진행 중 (3)                 │    │
│  │  ⠿ ▶ ● 아침 루틴 [매일]    │    │  ← 아코디언 섹션 (드래그 핸들 + 펼침/접힘)
│  │  ⠿ ▶ ● 식단     [매일]    │    │

│  │ [+ 새 섹션 추가]            │    │

│  └─────────────────────────────┘    │
├─────────────────────────────────────┤
│  2026년 4월 3일                     │  ← 선택 날짜 상세 패널
│  ┌─ 🍽️ 식단 계획 ──────────┐      │
│  │  🌅 아침                  │      │
│  │  ⠿ ☑ 현미밥         [✕]  │      │  ← 드래그 + 체크 + 삭제
│  │  [음식 입력...]  [추가]    │      │
│  │  ☀️ 점심                  │      │
│  │  ...                      │      │
│  └────────────────────────────┘      │
│  ┌─ 📝 메모 ─────────────────┐      │
│  │  [오늘의 메모...]          │      │
│  │  [저장]  저장됐어요!       │      │
│  └────────────────────────────┘      │
└─────────────────────────────────────┘
```

#### 구현된 기능 상세

**1. 월간 달력**
- 7열(일~토) x 필요 행 수의 그리드 렌더링
- 오늘 날짜: 보라색 테두리 + 볼드 텍스트
- 선택된 날짜: 보라색 반투명 배경 + 테두리
- 이전/다음 월 날짜: 흐리게 표시 (opacity 0.4)
- 월 이동 시 자동으로 1일 또는 오늘(해당 월인 경우) 선택

**2. 날짜별 인디케이터**
- **섹션 색상 점**: 해당 날짜에 활성화된 섹션마다 작은 원형 색상 점 표시 (하단)
- **식단 별표 ★**: 식단이 등록된 날짜에 왼쪽 상단 오렌지색 별 표시
- 메모+식단 동시 존재 시 그라데이션 인디케이터 (틸→오렌지)

**3. 날짜 상세 패널**
- 날짜 클릭 시 하단에 상세 패널 표시
- 패널 내용: 식단 계획 + 메모 입력
- 선택 날짜가 없으면 자동으로 오늘 또는 1일 선택

**4. 섹션(루틴 카테고리) 관리**

*추가 (섹션 추가 모달)*
- 섹션 이름 입력
- 8가지 색상 중 선택 (원형 버튼, 선택 시 흰 테두리)
- 반복 주기 선택 (7종): 매일, 주중(월~금), 주말(토~일), 매주(요일 지정), 격주(요일 지정), 격일, 매월(날짜 지정)
  - 매주/격주 선택 시: 요일 드롭다운 (일~토) 추가 표시
  - 매월 선택 시: 날짜 드롭다운 (1~31일) 추가 표시
- 시작일 설정 (date picker, 기본값: 오늘)
- 종료일 설정 (date picker, "무기한" 체크박스 → 기본 체크됨 → 종료일 입력 비활성)
- Enter 키로 저장 가능

*수정 (섹션 수정 모달)*
- 기존 값 프리필 (이름, 색상, 반복 주기, 요일/날짜, 시작일, 종료일)
- 저장 시 **범위 선택 모달** 호출 → "모든 날짜" 또는 "오늘부터" 선택
  - **모든 날짜**: 해당 섹션의 속성을 직접 수정
  - **오늘부터**: 기존 섹션의 종료일을 어제로 설정 + 수정된 내용으로 새 섹션 생성 (과거 기록 보존)

*삭제*
- 범위 선택 모달 호출
  - **모든 날짜**: 섹션 완전 삭제
  - **오늘부터**: 기존 섹션의 종료일을 어제로 설정 (과거 기록 보존)

**5. 항목 관리**

*인라인 추가 (아코디언 내부)*
- 이모지 입력 (2자 제한, 기본값 📌) + 라벨 입력
- Enter 키 또는 추가 버튼으로 저장
- 추가 후 자동 포커스 유지

*모달 추가/수정*
- 섹션 선택 드롭다운 + 이모지 + 라벨 입력
- 수정 시 범위 선택 모달 연동
  - **모든 날짜**: 항목 직접 수정
  - **오늘부터**: 기존 섹션을 어제까지로 끊고, 수정된 항목으로 새 섹션 생성

*삭제*
- 범위 선택 모달 연동
  - **모든 날짜**: 항목 직접 삭제
  - **오늘부터**: 기존 섹션을 어제까지로 끊고, 해당 항목 제외한 새 섹션 생성

**6. 식단 계획**
- 4가지 끼니별(🌅 아침, ☀️ 점심, 🌙 저녁, 🍎 간식) 독립 관리
- 텍스트 입력 + 추가 버튼 (또는 Enter 키)
- 각 음식 항목: 드래그 핸들(⠿) + 체크박스 + 라벨 + 삭제(✕)
- 체크박스: 식단 실행 여부 토글 (dailyRecords에 저장)
- 드래그앤드롭으로 같은 끼니 내 순서 변경

**7. 메모**
- 날짜별 텍스트 메모 — 체크리스트 탭 달성률 하단 + 캘린더 탭 날짜 상세 패널 양쪽에서 편집 가능
- 체크리스트 탭: 500ms 디바운스 자동 저장, "저장됨 ✓" 표시
- 캘린더 탭: 저장 버튼 클릭 시 저장, "저장됐어요!" 표시 (1.5초 후 페이드 아웃)
- 양쪽 모두 동일한 `data.memos[dateKey]` 데이터 공유

**8. 드래그앤드롭** (Pointer Events API 기반)

3가지 드래그 대상이 독립적으로 구현됨:

| 대상 | 드래그 핸들 | 동작 |
|------|-------------|------|
| 섹션 순서 | 아코디언 헤더의 `⠿` | `data.sections` 배열 순서 변경 |
| 항목 순서 | 아코디언 내부 항목의 `⠿` | `section.items` 배열 순서 변경 |
| 식단 순서 | 식단 항목의 `⠿` | `mealPlans[dateKey][mealType]` 배열 순서 변경 |

공통 구현 패턴:
- `pointerdown` → 고스트 요소 생성 (원본 복제, fixed 위치, 투명도 0.8)
- `pointermove` → 고스트 위치 갱신 + 드롭 대상에 `drag-over` 클래스 (상단 보라 라인)
- `pointerup` → 고스트 제거 + 배열 splice로 순서 변경 + 리렌더링

**9. 아코디언 섹션 관리 패널**
- 섹션을 상태별 그룹화:
  - **진행 중**: startDate ≤ 오늘 & (endDate 없음 또는 endDate ≥ 오늘)
  - **예정**: startDate > 오늘
  - **종료됨**: endDate < 오늘 (기본 숨김, "종료된 루틴 보기" 토글로 표시, opacity 0.5)
- 각 섹션 헤더: 드래그 핸들 + 펼침 화살표(▶/▼) + 색상 점 + 제목 + 반복 배지 + 날짜 범위 + 항목 수 + 수정/삭제 버튼
- 펼침 시: 항목 목록 + 인라인 추가 폼 표시

#### 구현된 기능 상세

**1. 날짜 네비게이션**
- 좌우 화살표로 ±1일 이동
- 날짜 형식: `2026년 4월 3일 (금)`

**2. 통계 카드**
- **전체 달성률** (보라): (체크리스트 완료 + 식단 완료) / (체크리스트 전체 + 식단 전체) × 100
- **루틴 달성률** (틸): 체크리스트 완료 / 체크리스트 전체 × 100
- **식단 달성률** (오렌지): 식단 완료 / 식단 전체 × 100

**3. 체크리스트 비교 테이블**
- 테이블 헤더: 카테고리 | 항목 | 계획 | 실행 | 상태
- 카테고리: 섹션 색상 점 + 섹션명 (해당 색상 텍스트)
- 계획: 항상 ✓ (계획에 포함된 항목이므로)
- 실행: 원형 체크 버튼 (클릭으로 토글 가능)
  - 체크 시 섹션 색상 배경 적용
- 상태: ✅ (완료) 또는 ❌ (미완료)
- 보라색 테마 테이블 스타일 (border, header 배경)

**4. 식단 비교 테이블**
- 테이블 헤더: 끼니 | 계획한 식단 | 실행 | 상태
- 끼니: 이모지 + 한글명 (오렌지 색상)
- 실행: 원형 체크 버튼 (클릭으로 토글 가능)
  - 체크 시 오렌지 배경 적용
- 상태: ✅ 또는 ❌
- 오렌지 테마 테이블 스타일 (별도 CSS 클래스 `tracker-meal`)

**5. 데이터 소스 분기**
- **당일 (`todayKey()`)**: 라이브 `completions` + `mealPlans` 데이터 사용
  - 체크 토글 가능 + 체크리스트 페이지와 양방향 동기화
- **과거 날짜**:
  - 먼저 `getSectionsForDate()`로 현재 섹션 기반 데이터 조회
  - 섹션이 없으면 `dailyRecords` 스냅샷에서 아카이브 데이터 로드
  - 아카이브 데이터는 토글 불가 (읽기 전용)
- 데이터 없는 날짜: "이 날짜에 기록된 데이터가 없습니다." 안내

| 키 | 한글명 | repeatDay 필요 | 판별 로직 |

| `everyOtherDay` | 격일 | X | 시작일 기준 짝수 번째 날 |
| `monthly` | 매월 | O (날짜 1-31) | getDate() === repeatDay |

## 6. 기술적 참고사항

### 날짜 포맷
- 내부 dateKey: `YYYY-M-D` (패딩 없음, 예: `2026-4-3`)
- ISO 변환 함수 존재: `dateKeyToISO()` / `isoToDateKey()` (date picker 연동용)
- 비일관적 패딩으로 인해 문자열 비교 시 주의 필요

### 데이터 저장
- **주 저장소**: SQLite — Docker: `/data/planner.db` (볼륨), 로컬: `DB_DIR` 환경변수 경로
- **로컬 캐시**: localStorage (`planner_v4_{username}`) — 오프라인 폴백용
- **저장 흐름**: saveData() → localStorage 즉시 저장 + 300ms 디바운스 서버 PUT
- **페이지 닫기**: navigator.sendBeacon() POST로 확실한 전송
- 서버 저장 실패 시 경고 배너 표시 (`#storage-warning`)
- 서버 측 데이터 크기 제한: 1MB

### 서버 바인딩
- **Docker 환경** (`DB_DIR=/data`): `0.0.0.0`에 바인딩 — Docker 내부 네트워크에서 tunnel 컨테이너 접근 허용
- **로컬 환경**: `127.0.0.1`에 바인딩 — 외부 접근 차단 (보안)
- `BIND_HOST` 환경변수로 직접 오버라이드 가능
- Docker에서 `0.0.0.0`은 격리된 Docker 가상 네트워크 내부에서만 노출되므로 보안상 안전

### 아키텍처 문서
- `architecture.html`: 프로젝트 전체 파이프라인을 시각적으로 표현한 인터랙티브 HTML 문서
- 서버/프론트엔드 흐름을 클릭 가능한 다이어그램으로 구성
- 각 단계 클릭 시 상세 설명 아코디언 펼침 (기술 스택, 관련 파일, 코드 흐름)
- 외부 의존성 없이 순수 CSS/JS로 구현, 브라우저에서 직접 열기 가능

### 접근성
- ARIA 레이블 미적용
- 키보드 네비게이션 부분 지원 (Enter 키 저장만)
- 스크린 리더 미지원

### 브라우저 호환
- ES6+ 문법 사용 (async/await, Promise, 템플릿 리터럴, 화살표 함수)
- Pointer Events API 사용 (IE 미지원)
- CSS 변수, flexbox, grid 사용
- 별도 트랜스파일링/폴리필 없음
